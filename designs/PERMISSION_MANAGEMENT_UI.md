# 权限管理可视化 UI 设计稿

## 设计概述
为 OpenClaw-Admin 平台设计权限管理功能的用户界面，提供直观、安全的 RBAC 权限管理体系可视化。

---

## 1. 功能需求

### 1.1 核心功能
- 角色管理（创建、编辑、删除）
- 权限分配（细粒度权限控制）
- 用户角色分配
- 权限继承与层级
- 权限审计日志
- 批量权限操作

### 1.2 用户场景
1. 管理员创建新角色并分配权限
2. 为用户分配/修改角色
3. 查看角色的权限详情
4. 审计权限变更历史
5. 批量修改用户权限

## 3. 组件规范

### 3.1 角色卡片
- **样式**: 带边框卡片，显示角色名称、权限数、用户数
- **颜色**: 根据角色级别使用不同边框颜色
- **操作**: 悬停显示编辑/删除按钮

### 3.2 权限复选框组
- **分组**: 按功能模块分组
- **折叠**: 支持展开/折叠模块
- **全选**: 支持模块级全选/全不选
- **联动**: 父级选中自动选中子级

### 3.3 角色选择器
- **双栏布局**: 可用角色/已分配角色
- **拖拽**: 支持拖拽分配
- **批量**: 支持批量添加/移除

## 4. 交互设计

### 4.1 角色管理流程
1. 点击"新建角色"
2. 输入角色名称和描述
3. 配置权限（勾选权限项）
4. 保存角色
5. 角色创建成功

### 4.2 用户分配流程
1. 进入用户详情页
2. 点击"角色管理"
3. 从可用角色列表选择
4. 点击"添加"按钮
5. 保存分配

### 4.3 权限变更确认
- 重要权限变更需要二次确认
- 显示变更影响范围
- 记录变更原因

## 5. 技术实现建议

### 5.1 前端组件结构
```vue
<template>
  <div class="permission-management">
    <!-- 角色列表 -->
    <role-list
      :roles="roles"
      @edit="handleEditRole"
      @delete="handleDeleteRole"
    />
    
    <!-- 角色编辑弹窗 -->
    <role-editor
      v-if="showEditor"
      :role="currentRole"
      @save="handleSaveRole"
      @cancel="showEditor = false"
    />
    
    <!-- 权限树 -->
    <permission-tree
      :permissions="allPermissions"
      v-model:selected="selectedPermissions"
    />
  </div>
</template>
```

### 5.2 API 接口设计
```javascript
// 获取角色列表
GET /api/roles

// 创建角色
POST /api/roles
{
  "name": "新角色",
  "description": "角色描述",
  "permissions": ["perm_001", "perm_002"]
}

// 更新角色
PUT /api/roles/:id
{
  "name": "更新后名称",
  "permissions": ["perm_001", "perm_003"]
}

// 分配用户角色
POST /api/users/:userId/roles
{
  "roleIds": ["role_001", "role_002"]
}

// 权限审计日志
GET /api/audit/permissions
{
  "startDate": "2026-04-01",
  "endDate": "2026-04-11"
}
```

## 6. 安全设计

### 6.1 权限保护
- 超级管理员角色不可删除
- 至少保留一个管理员角色
- 权限变更需要管理员密码验证

### 6.2 审计追踪
- 所有权限变更记录日志
- 记录变更前后对比
- 支持审计日志导出

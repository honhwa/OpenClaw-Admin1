# 批量操作功能 UI 设计稿

## 设计概述
为 OpenClaw-Admin 平台设计批量操作功能的用户界面，支持用户高效地对多条记录进行批量处理。

---

## 1. 功能需求

### 1.1 核心功能
- 多选记录（复选框）
- 全选/取消全选
- 批量删除
- 批量状态变更
- 批量导出
- 批量分配/转移

### 1.2 用户场景
1. 管理员需要删除多个过期任务
2. 批量更新多个用户的状态
3. 一次性导出多条审计日志
4. 批量分配任务给多个成员

## 3. 组件规范

### 3.1 复选框组件
- **位置**: 列表第一列
- **样式**: 标准复选框，选中时高亮显示
- **交互**: 点击可选择单条记录

### 3.2 批量操作按钮组
- **位置**: 列表上方固定位置
- **样式**: 主要操作使用主色调，危险操作使用红色
- **状态**: 未选择任何记录时禁用

### 3.3 选择计数显示
- **格式**: "已选择 N 项"
- **位置**: 批量操作栏左侧
- **动态更新**: 选择/取消选择时实时更新

---

## 4. 交互流程

### 4.1 批量删除流程
1. 用户勾选需要删除的记录
2. 点击"批量删除"按钮
3. 弹出确认对话框，显示选中记录列表
4. 用户确认或取消
5. 执行删除操作，显示进度提示
6. 操作完成，刷新列表

### 4.2 批量状态变更流程
1. 用户勾选需要变更的记录
2. 点击"批量状态变更"下拉菜单
3. 选择目标状态
4. 弹出确认对话框
5. 用户确认
6. 执行变更，显示成功/失败统计

## 5. 技术实现建议

### 5.1 前端组件结构
```vue
<template>
  <div class="batch-operation-bar" v-if="selectedCount > 0">
    <span class="selection-count">已选择 {{ selectedCount }} 项</span>
    <button @click="handleBatchDelete">批量删除</button>
    <el-dropdown @command="handleBatchStatusChange">
      <button>批量状态变更</button>
      <el-dropdown-menu>
        <el-dropdown-item v-for="status in statuses" :key="status" :command="status">
          {{ status }}
        </el-dropdown-item>
      </el-dropdown-menu>
    </el-dropdown>
  </div>
</template>
```

### 5.2 API 接口设计
```javascript
// 批量删除
POST /api/batch/delete
{
  "resource": "tasks",
  "ids": [1, 2, 3, 4, 5]
}

// 批量状态变更
POST /api/batch/update-status
{
  "resource": "tasks",
  "ids": [1, 2, 3],
  "status": "completed"
}
```

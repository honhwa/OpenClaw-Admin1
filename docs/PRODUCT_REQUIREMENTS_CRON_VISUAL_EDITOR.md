## 1. 项目概述

### 1.1 项目背景
OpenClaw Admin 系统需要增强 Cron 定时任务的配置体验。当前用户需要手动编写 Cron 表达式，存在以下痛点：
- Cron 表达式语法复杂，容易配置错误
- 缺乏实时预览和验证机制
- 常用任务需要重复配置，效率低下
- 无法直观查看任务执行历史

### 1.2 项目目标
为 OpenClaw Admin 提供**可视化 Cron 编辑器**，降低配置门槛，提升用户体验：
- ✅ 通过可视化界面配置定时任务
- ✅ 实时预览和校验 Cron 表达式
- ✅ 内置常用任务模板
- ✅ 完整的执行历史追踪

### 1.3 成功指标
- Cron 表达式配置错误率降低 **80%**
- 任务配置时间缩短 **60%**
- 用户满意度提升至 **90%+**

## 2. 需求规格

### 2.1 功能需求

#### FR-01: 可视化 Cron 表达式编辑器
**描述**: 提供分钟、小时、日期、月份、星期的可视化选择器

**验收标准**:
- [ ] 支持 5 个维度的独立配置（分钟/小时/日/月/周）
- [ ] 实时显示生成的 Cron 表达式
- [ ] 支持通配符 `*`、间隔 `*/n`、范围 `1-5`、列表 `1,3,5`
- [ ] 语法错误时实时提示具体原因
- [ ] 支持复制生成的 Cron 表达式

**优先级**: P1（高）

#### FR-02: 常用频率快捷按钮
**描述**: 提供一键设置常用频率的快捷按钮

**验收标准**:
- [ ] 每分钟
- [ ] 每小时
- [ ] 每天（指定时间）
- [ ] 每周（指定星期和时间）
- [ ] 每月（指定日期和时间）
- [ ] 自定义

**优先级**: P1（高）

#### FR-03: 任务模板库
**描述**: 内置常用任务模板，支持一键应用

**验收标准**:
- [ ] 内置模板列表：
  - 每日晨报（08:00）
  - 每小时健康巡检
  - 每日数据备份（02:00）
  - 每周日志清理（周日 03:00）
  - 每月报表生成（1 号 09:00）
- [ ] 模板支持自定义参数
- [ ] 支持保存自定义模板
- [ ] 模板支持导入/导出

**优先级**: P1（高）

#### FR-04: 任务执行历史查看
**描述**: 查看定时任务的执行历史记录

**验收标准**:
- [ ] 显示任务名称、执行时间、状态、输出结果
- [ ] 支持按任务名称筛选
- [ ] 支持按时间范围筛选
- [ ] 支持按状态筛选（成功/失败）
- [ ] 点击可查看单次执行的详细日志
- [ ] 失败任务显示错误堆栈信息
- [ ] 支持分页（每页 20/50/100 条）

**优先级**: P1（高）

#### FR-05: 任务管理
**描述**: 完整的任务 CRUD 操作

**验收标准**:
- [ ] 创建新任务（支持从模板创建）
- [ ] 编辑现有任务
- [ ] 删除任务（二次确认）
- [ ] 启用/禁用任务
- [ ] 立即执行一次任务（测试）
- [ ] 任务列表排序和筛选

**优先级**: P1（高）

#### FR-06: 实时执行时间预览
**描述**: 预览 Cron 表达式的未来执行时间

**验收标准**:
- [ ] 显示未来 10 次执行时间
- [ ] 支持切换时区显示
- [ ] 执行时间变更时实时更新

**优先级**: P2（中）

### 2.2 非功能需求

#### NFR-01: 性能要求
- 页面加载时间 < 2 秒
- Cron 表达式校验响应时间 < 100ms
- 执行历史查询响应时间 < 1 秒

#### NFR-02: 兼容性要求
- 支持 Chrome 90+、Firefox 88+、Safari 14+
- 支持 1920x1080、1366x768、1440x900 分辨率

#### NFR-03: 安全性要求
- 任务配置需权限验证
- 执行日志不包含敏感信息
- API 接口需身份认证

## 4. API 接口设计

### 4.1 任务管理接口

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/cron/tasks` | 获取任务列表 |
| POST | `/api/cron/tasks` | 创建任务 |
| GET | `/api/cron/tasks/:id` | 获取任务详情 |
| PUT | `/api/cron/tasks/:id` | 更新任务 |
| DELETE | `/api/cron/tasks/:id` | 删除任务 |
| POST | `/api/cron/tasks/:id/execute` | 立即执行任务 |
| PATCH | `/api/cron/tasks/:id/toggle` | 启用/禁用任务 |

### 4.2 模板接口

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/cron/templates` | 获取模板列表 |
| POST | `/api/cron/templates` | 创建自定义模板 |
| GET | `/api/cron/templates/:id` | 获取模板详情 |
| PUT | `/api/cron/templates/:id` | 更新模板 |
| DELETE | `/api/cron/templates/:id` | 删除模板 |

### 4.3 执行历史接口

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/cron/tasks/:id/history` | 获取执行历史 |
| GET | `/api/cron/history/:id/detail` | 获取执行详情 |
| DELETE | `/api/cron/history` | 清理历史数据 |

## 5. 数据库设计

### 5.1 任务表 (cron_tasks)

| 字段 | 类型 | 描述 |
|------|------|------|
| id | INTEGER PK | 任务 ID |
| name | VARCHAR(100) | 任务名称 |
| description | TEXT | 任务描述 |
| cron_expression | VARCHAR(50) | Cron 表达式 |
| command | VARCHAR(500) | 执行命令 |
| template_id | INTEGER | 模板 ID（可选） |
| status | VARCHAR(20) | 状态：enabled/disabled |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |
| last_executed_at | DATETIME | 最后执行时间 |
| next_executed_at | DATETIME | 下次执行时间 |

### 5.2 执行历史表 (cron_execution_history)

| 字段 | 类型 | 描述 |
|------|------|------|
| id | INTEGER PK | 记录 ID |
| task_id | INTEGER FK | 任务 ID |
| executed_at | DATETIME | 执行时间 |
| status | VARCHAR(20) | 状态：success/failed |
| output | TEXT | 执行输出 |
| error_message | TEXT | 错误信息 |
| duration_ms | INTEGER | 执行时长（毫秒） |

### 5.3 模板表 (cron_templates)

| 字段 | 类型 | 描述 |
|------|------|------|
| id | INTEGER PK | 模板 ID |
| name | VARCHAR(100) | 模板名称 |
| description | TEXT | 模板描述 |
| cron_expression | VARCHAR(50) | Cron 表达式 |
| command_template | TEXT | 命令模板（支持参数） |
| is_builtin | BOOLEAN | 是否内置模板 |
| created_at | DATETIME | 创建时间 |

 * * * * *

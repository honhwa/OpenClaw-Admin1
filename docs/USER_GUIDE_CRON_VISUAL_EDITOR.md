## 功能概述

Cron 可视化编辑器是 OpenClaw Admin 为定时任务管理提供的可视化配置工具，旨在降低 Cron 表达式的学习成本，让非技术人员也能轻松创建定时任务。

### 核心特性

- 🎨 **双模式编辑**：简单模式（模板选择）+ 高级模式（可视化字段配置）
- 🔍 **实时预览**：即时显示未来 5 次执行时间
- ✅ **智能校验**：实时验证 Cron 表达式有效性
- 📋 **任务模板**：内置常用任务模板（晨报、健康检查、数据备份等）
- 🗂️ **任务管理**：创建、编辑、删除、立即执行任务
- 📊 **执行历史**：查看任务执行记录与结果

### 适用场景

- 每日定时数据备份
- 每周系统健康检查
- 每月报告生成
- 定时消息推送
- 定期清理临时文件
- 定时 API 调用

## 快速开始

### 1. 进入 Cron 管理页面

登录 OpenClaw Admin 后，在左侧导航栏点击 **"任务计划 (Cron)"** 进入管理页面。

### 2. 创建新任务

点击页面右上角的 **"+ 新建任务"** 按钮。

### 3. 选择编辑模式

系统提供两种编辑模式：

#### 简单模式（推荐新手）

1. 选择 **"简单模式"** 标签
2. 从模板列表中选择预设任务类型
3. 配置任务名称、描述和执行间隔
4. 点击 **"保存"** 完成创建

#### 高级模式（灵活配置）

1. 选择 **"高级模式"** 标签
2. 使用可视化字段配置 Cron 表达式
3. 查看实时执行时间预览
4. 点击 **"保存"** 完成创建

### 4. 管理任务

创建的任务会显示在任务列表中，您可以：

- **查看**：点击任务名称查看详情
- **编辑**：点击编辑按钮修改任务配置
- **立即执行**：点击运行按钮手动触发任务
- **删除**：点击删除按钮移除任务
- **启用/禁用**：切换任务开关状态

## 使用指南

### 简单模式详解

简单模式适合不熟悉 Cron 表达式的用户，提供常用任务模板：

| 模板名称 | 执行频率 | 适用场景 |
|---------|---------|---------|
| 每日晨报 | 每天 8:00 | 每日工作开始时的报告推送 |
| 健康检查 | 每天 9:00 | 系统健康状态检查 |
| 数据备份 | 每天 2:00 | 数据库和文件备份 |
| 日志清理 | 每周日 3:00 | 清理过期日志文件 |
| 周报生成 | 每周一 10:00 | 自动生成周工作报告 |
| 月度统计 | 每月 1 日 9:00 | 月度数据统计分析 |

**操作步骤：**

1. 选择任务模板
2. 修改任务名称（可选）
3. 调整执行时间（可选）
4. 填写任务描述（可选）
5. 点击保存

### 高级模式详解

高级模式提供完整的 Cron 表达式配置能力：

#### 字段说明

| 字段 | 允许值 | 特殊字符 | 说明 |
|-----|-------|---------|-----|
| 分钟 | 0-59 | `* , - /` | 任务执行的分钟 |
| 小时 | 0-23 | `* , - /` | 任务执行的小时 |
| 日期 | 1-31 | `* , - / ?` | 每月的日期 |
| 月份 | 1-12 | `* , - /` | 月份 |
| 星期 | 0-6 (0=周日) | `* , - / ?` | 星期几 |

**特殊字符含义：**

- `*` - 任意值（每分钟/每小时/每天等）
- `,` - 列举多个值（如 `1,3,5` 表示第 1、3、5 分钟）
- `-` - 指定范围（如 `1-5` 表示 1 到 5）
- `/` - 指定增量（如 `*/15` 表示每 15 分钟）
- `?` - 不指定值（用于日期或星期字段）

#### 配置示例

**示例 1：每天上午 9:30 执行**
- 分钟：`30`
- 小时：`9`
- 日期：`*`
- 月份：`*`
- 星期：`?`
- 表达式：`30 9 * * ?`

**示例 2：每周一和周五下午 5:00 执行**
- 分钟：`0`
- 小时：`17`
- 日期：`?`
- 月份：`*`
- 星期：`1,5`
- 表达式：`0 17 ? * 1,5`

**示例 3：每 15 分钟执行一次**
- 分钟：`*/15`
- 小时：`*`
- 日期：`*`
- 月份：`*`
- 星期：`?`
- 表达式：`*/15 * * * ?`

**示例 4：每月最后一天下午 6:00 执行**
- 分钟：`0`
- 小时：`18`
- 日期：`L`
- 月份：`*`
- 星期：`?`
- 表达式：`0 18 L * ?`

## Cron 表达式基础

### 标准 Cron 格式

```
┌───────────── 分钟 (0 - 59)
│ ┌───────────── 小时 (0 - 23)
│ │ ┌───────────── 日期 (1 - 31)
│ │ │ ┌───────────── 月份 (1 - 12)
│ │ │ │ ┌───────────── 星期 (0 - 6) (0=周日)
│ │ │ │ │
* * * * *
```

### 常用表达式参考

| 表达式 | 说明 |
|-------|-----|
| `* * * * *` | 每分钟执行 |
| `*/5 * * * *` | 每 5 分钟执行 |
| `0 * * * *` | 每小时执行 |
| `0 0 * * *` | 每天午夜执行 |
| `0 0 * * 0` | 每周日凌晨执行 |
| `0 0 1 * *` | 每月第一天执行 |
| `0 0 1 1 *` | 每年 1 月 1 日执行 |
| `0 9 * * 1-5` | 工作日每天上午 9 点执行 |
| `0 0 0 1,15 *` | 每月 1 日和 15 日执行 |

### 特殊符号详解

#### 星号 (*)
表示"任意值"，在分钟字段表示"每分钟"，在小时字段表示"每小时"。

#### 逗号 (,)
用于列举多个值：
- `1,3,5` - 第 1、3、5 分钟
- `MON,WED,FRI` - 周一、周三、周五

#### 连字符 (-)
用于指定范围：
- `1-5` - 第 1 到第 5 分钟
- `9-17` - 上午 9 点到下午 5 点

#### 斜杠 (/)
用于指定增量：
- `*/15` - 每 15 分钟（0,15,30,45）
- `0-30/5` - 从 0 到 30，每 5 分钟（0,5,10,15,20,25,30）

#### 问号 (?)
仅用于日期和星期字段，表示"不指定值"。当其中一个字段被指定时，另一个应使用 `?`。

#### 字母 L
表示"最后"：
- `L` - 月份的最后一天
- `5L` - 每月的最后一个星期五

#### 字母 W
表示"最近的平日"：
- `15W` - 离 15 号最近的平日（周一至周五）

#### 字母 #
用于指定第几个星期几：
- `FRI#3` - 每月的第三个星期五

## 常见问题

### Q1: Cron 表达式验证失败怎么办？

**A:** 检查以下几点：
1. 确保所有字段值在允许范围内
2. 日期和星期字段不能同时指定具体值（应使用 `?`）
3. 特殊字符使用正确（`,`、`-`、`/`、`?`）
4. 表达式格式正确（5 个或 6 个字段）

### Q2: 任务没有按时执行？

**A:** 可能的原因：
1. OpenClaw Admin 服务未运行
2. 任务被禁用（开关关闭）
3. 服务器时间与时区设置不正确
4. 系统负载过高导致任务延迟

### Q3: 如何查看任务的详细执行日志？

**A:** 在任务列表中找到对应任务，点击"执行历史"按钮，可以查看每次执行的详细信息，包括：
- 执行时间
- 执行状态
- 输出内容
- 错误信息

### Q4: 可以导入/导出任务配置吗？

**A:** 当前版本支持：
- 导出单个任务配置为 JSON 文件
- 批量导出所有任务配置
- 从 JSON 文件导入任务配置（需管理员权限）

### Q5: 任务执行失败如何通知？

**A:** 系统支持多种通知方式：
- 站内消息通知
- 邮件通知（需配置 SMTP）
- Webhook 回调
- 集成到 IM 工具（飞书、钉钉等）

## 附录

### 附录 A: 时区设置

OpenClaw Admin 默认使用服务器时区。如需修改：

1. 进入 **"系统设置"**
2. 找到 **"时区配置"**
3. 选择目标时区
4. 保存设置

### 附录 B: 任务优先级

系统支持任务优先级设置：
- **高优先级**：优先执行，适用于关键任务
- **中优先级**：默认优先级
- **低优先级**：系统空闲时执行

### 附录 C: 任务依赖

高级功能：支持设置任务依赖关系
- 任务 B 依赖任务 A：任务 A 成功后才执行任务 B
- 任务 B 依赖任务 A：任务 A 完成后（无论成功失败）执行任务 B

### 附录 D: 性能建议

- 避免创建过于频繁的任务（如每秒执行）
- 长时间运行的任务建议使用异步执行
- 定期清理执行历史，避免数据库过大
- 对于关键任务，建议设置监控告警
